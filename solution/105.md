## 1. Problem

105. Construct Binary Tree from Preorder and Inorder Traversal
    
## 2. Summary

* Preorder: root → left → right → first element is root
* Inorder: left → root → right → root splits inorder into left/right

## 3. Key Points





**Steps:**

1. Take `preorder[0]` as root
2. Find root index in `inorder` → split into left/right
3. Slice preorder according to left subtree size
4. Recurse for left and right
5. Return root

**Python Template:**

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def buildTree(self, preorder, inorder):
        if not preorder: return None
        root = TreeNode(preorder[0])
        i = inorder.index(preorder[0])
        root.left = self.buildTree(preorder[1:i+1], inorder[:i])
        root.right = self.buildTree(preorder[i+1:], inorder[i+1:])
        return root
```


## 4. Code Implementation:

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        if not preorder or not inorder:
            return None

        root_v = preorder[0]
        root_idx = inorder.index(root_v)
        root = TreeNode(root_v)

        root.left = self.buildTree(preorder[1:root_idx+1], inorder[0: root_idx])
        root.right = self.buildTree(preorder[root_idx+1:], inorder[root_idx+1:])

        return root   
```

## 8. Note

**Optimized (O(n)) Idea:**

* Build `val → index` dict for inorder to avoid repeated search
* Use pointers instead of slicing for less space

* **Mistakes Summary:**

1. Didn’t realize `preorder[0]` is always root.
2. Unclear how `inorder.index(root_val)` splits left/right.
3. Didn’t consider slicing preorder based on left subtree size.
4. Didn’t know value uniqueness is assumed.
5. Concerned about repeated `index()` but didn’t know dict optimization.
